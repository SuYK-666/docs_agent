# Open Source Release Checklist

## Repository setup

- [ ] Set repository visibility and description.
- [ ] Add topics and homepage URL.
- [ ] Protect `main` branch.

## Legal and policy

- [X] Add LICENSE.
- [X] Add CODE_OF_CONDUCT.
- [X] Add CONTRIBUTING guide.
- [X] Add SECURITY policy.

## Engineering hygiene

- [X] Add .gitignore for runtime and local artifacts.
- [X] Add CI compile workflow.
- [X] Add issue/PR templates.
- [ ] Verify no secrets are committed.
- [ ] Verify sample config values only.

## Docs readiness

- [X] Refresh README structure and run instructions.
- [X] Add badges and contributor section.
- [X] Add license and contribution links.

## First public release

- [ ] Create initial tag/release notes.
- [ ] Pin minimal supported Python version.
- [ ] Announce contribution workflow.
